Installation de services réseau, TP (2025)

Préambule

Cette SAÉ a pour objectif d’installer et configurer des services réseaux nécessaires au développement et au déploiement d’applications Web dynamiques. Elle s’appuie sur les ressources R2.04 et R2.05 et se déroule dans un environnement Linux, en utilisant Apache comme serveur Web, MySQL comme base de données, et PHP comme langage serveur.

Configuration et supervision du serveur Apache

Il est demandé de se familiariser avec l’environnement de travail fourni via une machine virtuelle, puis de vérifier le bon fonctionnement du serveur Apache. Les étapes incluent l’inspection du statut du service, l’identification des fichiers de configuration, la localisation de l'exécutable du serveur, et la détermination de sa version. Il faut également lister les modules actifs (statiques et dynamiques) et analyser certains modules spécifiques.

Sécurisation d’un dossier privé avec Apache

Une partie privée du serveur Web doit être mise en place dans un dossier protégé par un système d’authentification basique géré par Apache. Il s'agit de créer le dossier, une page HTML personnalisée, des liens de navigation, et de configurer l’accès restreint à l’aide d’un login et mot de passe.

Vérification de la prise en charge du module PHP

Il est demandé de vérifier si le module PHP est activé sur le serveur Web. Cela implique de créer un script de test avec la fonction phpinfo(), d’y accéder via un navigateur, et de relever plusieurs informations sur la configuration PHP. Il faut également déterminer où sont localisés les fichiers de configuration et comprendre comment les modifier.

Manipulations avancées du module PHP

Après avoir localisé l’exécutable du module PHP, il faut confirmer sa version, désactiver le module via la ligne de commande, observer les conséquences de cette désactivation, puis réactiver le module. Des tests avec des URL et le contenu de dossiers spécifiques doivent confirmer chaque changement d’état.

Interprétation du PHP dans des fichiers HTML

Il est demandé d'insérer du code PHP dans un fichier .html, de tester son interprétation par le serveur, puis de configurer Apache pour que tous les fichiers .html soient traités comme des fichiers PHP. Un test permettra de vérifier la bonne prise en compte de cette directive.
