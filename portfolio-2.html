Installation d'un poste pour le développement, Docker

Contexte

Cette SAÉ vise à apprendre aux étudiants à installer et configurer un poste de développement, non pas de manière basique, mais de façon professionnelle. Il ne s’agit pas simplement d’installer Visual Studio Code ou GCC. Le développement couvre de nombreux domaines comme la programmation système, le développement Web, mobile ou encore les bases de données. Il implique également des tâches connexes comme la documentation, la gestion de code source ou encore des conversions de fichiers. Il est donc essentiel de maîtriser l’installation d’outils, de bibliothèques et de services nécessaires à ces tâches.

Environnement de travail professionnel

L’objectif est de permettre aux futurs professionnels du développement de savoir comment mettre en place un environnement de travail propre, efficace, et compatible avec le travail en équipe. Cela signifie aussi être capable de reproduire rapidement une configuration fonctionnelle, ce qui est crucial, même dans de grandes structures où les équipes techniques gèrent normalement le matériel et les logiciels. Les développeurs conservent une certaine liberté de configuration et doivent en être responsables.

Notions d’image et de conteneur

Une image est un modèle qui permet de créer un conteneur, un peu comme un fichier ZIP que l’on décompresse dans un dossier. Elle contient une arborescence de fichiers et de dossiers, avec des programmes prêts à être exécutés. Un conteneur est une instance de cette image, mais avec une différence clé : il est "vivant" car il contient un ou plusieurs processus en cours d’exécution. Le conteneur est en fait ce processus, ce qui le rend bien plus dynamique qu’un simple dossier.

Mise en pratique avec Docker

Pour utiliser une image, il faut d’abord la récupérer depuis un dépôt. Sur les ordinateurs de l’IUT, un dépôt interne est utilisé à la place de Docker Hub pour des raisons de sécurité. La commande docker image pull hello-world permet de télécharger l’image localement. Ensuite, on crée un conteneur avec la commande docker container run hello-world, ce qui exécute un processus basé sur cette image. Si tout se passe bien, un message de confirmation s’affiche dans le terminal, indiquant que Docker fonctionne correctement.

Contexte de communication pour un client

Dans un second temps, vous travaillez pour un client qui lance un nouveau produit et souhaite organiser une présentation. Cette présentation sera accessible à la fois sur place et à distance via Teams. Une invitation doit être envoyée à des clients VIP, et vous êtes chargé de traiter ces invitations à partir d’un fichier CSV contenant leurs informations.

Format des données à traiter

Le fichier CSV contient plusieurs colonnes : prénom, nom, société, code d’invitation et adresse e-mail. Ces données peuvent être séparées par des virgules, des points-virgules, ou même des tabulations. Les champs peuvent parfois être entourés de guillemets. Il est donc important de savoir manipuler ce type de fichiers correctement pour automatiser la génération d’invitations.

Génération des invitations en PDF

L’objectif final est de produire des invitations sous forme de fichiers PDF, un format universel, lisible sur tout type d’appareil sans nécessiter de logiciel payant. Contrairement aux documents Word ou Excel, les PDF sont légers, non modifiables et adaptés à l’impression. C’est pourquoi ils sont privilégiés dans un contexte professionnel, notamment pour ce projet d’invitation client.
